# Content Syncer

将年月目录（如 `25/01/`、`26/03/`）中的日志文件整理到 `collections` 目录，按类型分类管理，便于检索和维护。

---

## 一、目录结构

```
log-master/
├── 25/                    # 年份目录（两位年份，如 25 = 2025）
│   └── 01/                # 月份目录（两位月份，如 01 = 1月）
│       ├── 01.md          # 日志文件（两位日期，如 01 = 1日）
│       └── 02.md
├── 26/
│   └── 03/
│       └── 15.md
└── collections/           # 分类整理目录
    ├── Css/               # CSS相关
    ├── Js/                # JavaScript相关
    ├── Git/               # Git相关
    ├── Software/          # 开发工具
    ├── Idea/              # 想法观点
    ├── Vue/               # Vue相关
    ├── React/             # React相关
    ├── RegExp/            # 正则表达式
    ├── Chrome/            # Chrome浏览器
    ├── Markdown/          # Markdown相关
    ├── Emoji/             # Emoji表情
    └── Log/               # 同步记录
```

### 命名规则

| 层级 | 格式 | 示例 | 说明 |
|-----|------|------|------|
| 年份 | 两位数字 | `25` | 2025年 |
| 月份 | 两位数字 | `03` | 3月 |
| 日期 | 两位数字 | `15` | 15日 |

---

## 二、分类规则

### 分类列表

| 分类 | 内容类型 | 目标文件 |
|-----|---------|---------|
| **Css** | CSS特性、属性、CSS Tricks | `collections/Css/index.md` |
| **Js** | JavaScript API、语言特性 | `collections/Js/index.md` |
| **Git** | Git命令、版本控制技巧 | `collections/Git/index.md` |
| **Software** | 开发工具、网站、插件、配置 | `collections/Software/index.md` |
| **Idea** | 想法、观点、知识、设计原则 | `collections/Idea/index.md` |
| **Vue** | Vue框架相关知识 | `collections/Vue/index.md` |
| **React** | React框架相关知识 | `collections/React/index.md` |
| **RegExp** | 正则表达式语法、用法 | `collections/RegExp/index.md` |
| **Chrome** | Chrome浏览器、DevTools | `collections/Chrome/index.md` |
| **Markdown** | Markdown语法、使用技巧 | `collections/Markdown/index.md` |
| **Emoji** | 表情符号、符号组合 | `collections/Emoji/index.md` |

### 分类区分准则

| 分类 | 定义 | 示例 |
|-----|------|------|
| **Software** | 可直接使用的工具、库、配置 | "VSCode插件推荐"、"axios使用" |

## 三、工作流程

### 步骤1：确定同步范围

| 同步类型 | 触发方式 | 处理范围 |
|---------|---------|---------|
| 指定同步 | "同步26年3月" | 处理 `26/03/` 下所有文件 |
| 指定同步 | "整理25年" | 处理 `25/` 下所有月份目录 |
| 全量同步 | 未指定范围 | 扫描所有年份目录，跳过已记录日期 |

### 步骤2：遍历日志文件

遍历目标目录下所有 `.md` 文件，例如：
- `26/03/15.md`
- `26/03/16.md`

### 步骤3：解析内容条目

**核心规则**：以 `#`、`##`、`###` 标题为界，每个标题到下一个标题之间为一个完整条目。

处理流程：
1. 读取文件全部内容
2. 按标题分割成独立条目
3. 跳过空条目（无标题或无实质内容）
4. 逐条判断分类归属

### 步骤4：去重检查

写入前执行三层去重检查：

| 检查类型 | 检查内容 | 处理方式 |
|---------|---------|---------|
| 链接去重 | URL是否已存在 | 跳过重复 |
| 标题去重 | 标题文字是否相同 | 跳过重复 |
| 内容去重 | 核心代码/描述是否重复 | 跳过重复 |

### 步骤5：分类写入

根据条目内容归入对应分类，追加写入 `collections/XXX/index.md`。

### 步骤6：处理非MD文件

直接复制到对应目录，不在 `index.md` 中添加引用：

| 文件类型 | 目标目录 |
|---------|---------|
| `.js` | `collections/Js/` |
| `.css` | `collections/Css/` |
| `.less` | `collections/Css/` |
| `.json` | `collections/Software/` |
| 图片/媒体 | `collections/Software/assets/` |
| 其他代码 | 根据扩展名分类 |

### 步骤7：更新同步记录

更新 `collections/Log/todo.md`：
- 在最后一行追加本次同步的日期
- 格式：`YY/MM/DD`（如 `26/03/18`）
- 多日同步：`26/03/15 - 26/03/18`

### 步骤8：生成未分类报告

同步完成后，生成完整报告：

| 报告类型 | 内容说明 |
|---------|---------|
| 新增内容 | 本次同步新增到 collections 的条目 |
| 已存在 | 已在 collections 中无需更新的条目 |
| 跳过文件 | 内容为空或仅含 emoji 的文件 |
| 无法分类 | 无法归入现有分类的内容（需用户确认） |

---

## 四、强制要求

> **【最高优先级】** collections 内必须包含全部日志文件内容，不能缺少任何条目。禁止创建非日志文件的原创内容。

1. **单一分类**：每个条目只能归入一个分类
2. **完整报告**：同步完成后必须向用户报告新增、已存在、无法分类的内容
3. **用户确认**：无法分类的内容必须等待用户确认后再处理
4. **保持原始格式**：**【非常重要】** 严格保持原始内容的格式不变，包括：
   - 标题格式（链接在标题上或单独一行，原样保留）
   - 空行数量和位置
   - 缩进和对齐方式
   - 代码块格式
   - 引用格式
   - 列表格式
   - 绝对不能对原始内容做任何形式的调整或优化

---

## 五、创建新分类

当内容无法归入现有分类时：
1. 向用户报告无法分类的具体内容
2. 等待用户确认分类方式或创建新分类
3. 创建新分类步骤：
   - 在 `collections/` 下创建文件夹（如 `NewCat/`）
   - 创建 `index.md` 文件
   - 更新本 skill 的分类规则

---

## 六、注意事项

| 序号 | 注意事项 | 说明 |
|-----|---------|------|
| 1 | 逐条核对 | 必须读取原始文件逐条处理 |
| 2 | 完整条目 | 每个标题到下一个标题为完整整体 |
| 3 | 避免重复 | 写入前检查是否已存在 |
| 4 | 跳过空文件 | 内容为空的 md 文件直接跳过 |
| 5 | 批量优化 | 同一月份文件批量处理 |
| 6 | 标题兼容 | 支持 `#`、`##`、`###` 三种格式 |
| 7 | 特殊文件 | emoji-only 文件归入 Emoji 分类 |
| 8 | **保持原样** | **【关键】** 原始内容原样复制，不做任何格式调整！！ |
