# Content Syncer

将年月目录（如 `25/01/`、`26/03/`）中的日志文件整理到 `collections` 目录，按类型分类管理，便于检索和维护。

---

## 一、目录结构

```
log-master/
├── 25/                    # 年份目录（两位年份，如 25 = 2025）
│   └── 01/                # 月份目录（两位月份，如 01 = 1月）
│       ├── 01.md          # 日志文件（两位日期，如 01 = 1日）
│       └── 02.md
├── 26/
│   └── 03/
│       └── 15.md
└── collections/           # 分类整理目录
    ├── Css/               # CSS相关
    ├── Js/                # JavaScript相关
    ├── Git/               # Git相关
    ├── Software/          # 开发工具
    ├── Idea/              # 想法观点
    ├── Vue/               # Vue相关
    ├── React/             # React相关
    ├── RegExp/            # 正则表达式
    ├── Chrome/            # Chrome浏览器
    ├── Markdown/          # Markdown相关
    ├── Emoji/             # Emoji表情
    └── Log/               # 同步记录
```

### 命名规则

| 层级 | 格式 | 示例 | 说明 |
|-----|------|------|------|
| 年份 | 两位数字 | `25` | 2025年 |
| 月份 | 两位数字 | `03` | 3月 |
| 日期 | 两位数字 | `15` | 15日 |

## 三、工作流程

### 步骤1：确定同步范围

| 同步类型 | 触发方式 | 处理范围 |
|---------|---------|---------|
| 指定同步 | "同步26年3月" | 处理 `26/03/` 下所有文件 |
| 指定同步 | "整理25年" | 处理 `25/` 下所有月份目录 |

### 步骤2：遍历日志文件

遍历目标目录下所有 `.md` 文件，例如：
- `26/03/15.md`
- `26/03/16.md`

### 步骤3：解析内容条目

**【最高优先级规则】** 以 `#`、`##`、`###` 标题为界，每个标题到下一个标题之间为一个完整条目。

**【核心处理规则 - 必须严格遵守】**

| 规则编号 | 规则内容 | 优先级 | 说明 |
|---------|---------|-------|------|
| **R1** | **保持链接完整性** | ⭐⭐⭐⭐⭐（最高） | 标题中的 Markdown 链接 `[文本](url)` 必须完整保留在同一行，不得分离为多行，不得修改链接格式 |
| **R2** | 清除数字前缀 | ⭐⭐⭐⭐ | 仅移除标题前的序号前缀（如 `1、`、`2、`、`13、`），保持标题其他内容不变 |
| **R3** | 保持原始格式 | ⭐⭐⭐ | 标题中的链接、文本内容保持原样，不做任何修改 |

**处理流程**：
1. 读取文件全部内容
2. 按标题分割成独立条目
3. 跳过空条目（无标题或无实质内容）
4. **处理标题**：
   - 识别标题中的链接格式 `[文本](url)`
   - 保留链接的完整性，不破坏 `[文本](url)` 结构
   - 只移除数字前缀（如 `1、`、`2、`），不触碰链接部分
5. 保持原始内容格式不变
6. 逐条判断分类归属

#### 处理示例（正确 vs 错误）

| 原始内容 | 正确处理结果 | 错误处理结果（禁止） | 说明 |
|---------|-------------|---------------------|------|
| `### 13、[使用chrome浏览器直接调post接口](url)` | `### [使用chrome浏览器直接调post接口](url)` | `### 使用chrome浏览器直接调post接口\n> [url]` | 仅移除 `13、`，链接必须完整保留在标题内 |
| `### 5、[Vue.js技术揭秘](https://ustbhuangyi.github.io/vue-analysis/)` | `### [Vue.js技术揭秘](https://ustbhuangyi.github.io/vue-analysis/)` | `### Vue.js技术揭秘\n> [链接]` | 链接必须完整，不得拆分 |
| `### 2、禁止在对象实例上直接使用object.prototypes` | `### 禁止在对象实例上直接使用object.prototypes` | 同上（无链接） | 仅移除 `2、` |

#### 标题链接格式识别

必须完整识别并保留以下格式：
- `[链接文本](https://example.com)` - 标准链接
- `[链接文本](https://example.com/path?query=1)` - 带参数链接
- `[带空格的链接文本](https://example.com)` - 带空格的链接文本

**错误示例（禁止）**：
```markdown
### 13、使用chrome浏览器直接调post接口
> [使用chrome浏览器直接调post接口](url)
```

**正确示例（必须）**：
```markdown
### [使用chrome浏览器直接调post接口](url)
```

### 步骤4：去重检查

写入前执行三层去重检查（标题去重在清除数字前缀后进行）：

| 检查类型 | 检查内容 | 处理方式 |
|---------|---------|---------|
| 链接去重 | URL是否已存在 | 跳过重复 |
| 标题去重 | 清除前缀后的标题文字是否相同 | 跳过重复 |
| 内容去重 | 核心代码/描述是否重复 | 跳过重复 |

### 步骤5：分类写入

根据条目内容归入对应分类，追加写入 `collections/XXX/index.md`。

### 步骤6：处理非MD文件

直接复制到对应目录，不在 `index.md` 中添加引用：

| 文件类型 | 目标目录 |
|---------|---------|
| `.js` | `collections/Js/` |
| `.css` | `collections/Css/` |
| `.less` | `collections/Css/` |
| `.json` | `collections/Software/` |
| 图片/媒体 | `collections/Software/assets/` |
| 其他代码 | 根据扩展名分类 |

### 步骤7：更新同步记录

更新 `collections/Log/todo.md`：
- 在最后一行追加本次同步的日期
- 格式：`YY/MM/DD`（如 `26/03/18`）
- 多日同步：`26/03/15 - 26/03/18`

### 步骤8：生成同步报告

同步完成后，生成完整报告：

| 报告类型 | 内容说明 |
|---------|---------|
| 新增内容 | 本次同步新增到 collections 的条目 |
| 已存在 | 已在 collections 中无需更新的条目 |
| 跳过文件 | 内容为空或仅含 emoji 的文件 |
| 无法分类 | 无法归入现有分类的内容（需用户确认） |

## 四、强制要求（必须100%遵守）

> **【最高优先级】** collections 内必须包含全部日志文件内容，不能缺少任何条目。禁止创建非日志文件的原创内容。

| 序号 | 规则 | 优先级 | 说明 |
|-----|------|-------|------|
| 1 | **保持链接格式** | ⭐⭐⭐⭐⭐ | **【关键】** 标题中的 Markdown 链接 `[文本](url)` 必须保持在标题内，不得分离为多行，不得破坏链接结构 |
| 2 | **清除数字前缀** | ⭐⭐⭐⭐ | 标题前的 `1、`、`2、`、`3、` 等序号前缀应去除，使标题更简洁 |
| 3 | **保持原始格式** | ⭐⭐⭐ | 严格保持原始内容的格式不变，包括空行数量、代码块格式、引用格式等 |
| 4 | 单一分类 | ⭐⭐⭐ | 每个条目只能归入一个分类 |
| 5 | 完整报告 | ⭐⭐ | 同步完成后必须向用户报告新增、已存在、无法分类的内容 |
| 6 | 用户确认 | ⭐⭐ | 无法分类的内容必须等待用户确认后再处理 |

**优先级说明**：
- 规则1（保持链接格式）优先级最高，任何情况下都不得违反
- 规则2（清除数字前缀）与规则3（保持原始格式）冲突时，以规则2为准
- 规则1（保持链接格式）在任何情况下都优先于其他所有规则

---

## 五、创建新分类

当内容无法归入现有分类时：
1. 向用户报告无法分类的具体内容
2. 等待用户确认分类方式或创建新分类
3. 创建新分类步骤：
   - 在 `collections/` 下创建文件夹（如 `NewCat/`）
   - 创建 `index.md` 文件
   - 更新本 skill 的分类规则

## 六、注意事项

| 序号 | 注意事项 | 说明 |
|-----|---------|------|
| 1 | 逐条核对 | 必须读取原始文件逐条处理 |
| 2 | 完整条目 | 每个标题到下一个标题为完整整体 |
| 3 | 避免重复 | 写入前检查是否已存在 |
| 4 | 跳过空文件 | 内容为空的 md 文件直接跳过 |
| 5 | 批量优化 | 同一月份文件批量处理 |
| 6 | 标题兼容 | 支持 `#`、`##`、`###` 三种格式 |
| 7 | 特殊文件 | emoji-only 文件归入 Emoji 分类 |
| 8 | **链接完整性检查** | 写入前必须验证标题中的链接是否完整保留 |

## 八、错误处理与回退

如果处理过程中出现以下情况，立即停止并报告：
1. 意外破坏了链接格式
2. 不确定如何处理某个条目
3. 发现内容可能丢失的风险
